Apa itu Physical Model?

"Physical Model adalah representasi paling detail dari basis data yang menggambarkan bagaimana data benar-benar disimpan di dalam sistem penyimpanan (storage) komputer. Model ini fokus pada aspek teknis implementasi, seperti struktur tabel, indeks, tipe data, partisi, relasi antar tabel, serta mekanisme akses data. Kalau Conceptual Model menggambarkan apa saja yang ada (entitas, atribut, relasi), dan Logical Model menjelaskan bagaimana hubungan data disusun secara logis, maka Physical Model menjelaskan bagaimana data tersebut diimplementasikan dalam DBMS tertentu."

  1. Ciri-ciri Physical Model

    • Sangat teknis → menyesuaikan dengan DBMS yang digunakan (misalnya MySQL, Oracle, PostgreSQL).
    • Mendefinisikan detail struktur penyimpanan → tabel, kolom, tipe data, primary key, foreign key.
    • Menjelaskan optimasi kinerja → penggunaan indeks, clustering, partisi data.
    • Menjadi acuan implementasi → inilah model yang benar-benar diterjemahkan menjadi perintah SQL (DDL).

  2. Komponen Utama Physical Model

    • Tabel dan Kolom → lengkap dengan nama dan tipe data (contoh: VARCHAR(50), INT).
    • Kunci (Key) → Primary Key, Foreign Key, Unique Key.
    • Indeks → untuk mempercepat pencarian data.
    • Constraint → aturan seperti NOT NULL, CHECK, DEFAULT.
    • Storage Details → kadang termasuk cara data dipartisi atau ditempatkan dalam disk.

  3. Contoh singkat

    Dalam sistem kampus

    • Entitas: Mahasiswa, Dosen, Mata Kuliah.
    • Atribut: Mahasiswa punya NIM, Nama; Dosen punya NIP.
    • Relasi: Mahasiswa mengambil Mata Kuliah, Dosen mengajar Mata Kuliah.
